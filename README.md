# synthetic-panel — синтетическая исследовательская панель агентства

**Версия: v1.3 (июль 2026)**

Скилл для Claude Code и OpenAI Codex: тестирование маркетинговых гипотез на смоделированных профилях аудитории по опубликованной методологии SSR (Semantic Similarity Rating) + режим «Карта сегментов / AI CJM» (устойчивая сегментация аудитории, путь клиента, проверка ключевых сообщений).

Новое в v1.3: полная карточка профиля в генерации (все поля сегмента); якорные наборы v2, прошедшие вычислительный гейт монотонности на валидированном эмбеддере (`ai-forever/ru-en-RoSBERTa`); честная статистика сравнения (парный бутстреп, P(A>B), трёхуровневые ярлыки разделимости вместо сырых CI в клиентском слое); встроенный самоконтроль каждого прогона (плацебо-стимул + пара-ловушка + слепые метки — провал контролей помечает прогон непригодным); режимы `exploratory`/`validated` с фиксацией модели; секции «Главное» и «Что с этим делать», паспорт методологии; клиентский HTML-рендер (`scripts/render_client.py`).

**Важно про среды:** скилл работает в агентских средах, которые читают стандарт SKILL.md — **Codex CLI/cloud** и **Claude Code**. В веб-чате ChatGPT скиллы не исполняются: там останется ручная цепочка промптов; этот репозиторий — её воспроизводимая замена для Codex.

## Установка

Требования: Python 3.10+, доступ к PyPI и HuggingFace (первый запуск скачает embedding-модель).

**Codex** (скиллы ищутся в `$CWD/.agents/skills`, `$REPO_ROOT/.agents/skills`, `$HOME/.agents/skills`):
```bash
git clone https://github.com/kobozevomd/synthetic-panel.git ~/.agents/skills/synthetic-panel
bash ~/.agents/skills/synthetic-panel/scripts/setup.sh
```

**Claude Code** (скиллы ищутся в `~/.claude/skills`):
```bash
git clone https://github.com/kobozevomd/synthetic-panel.git ~/.claude/skills/synthetic-panel
bash ~/.claude/skills/synthetic-panel/scripts/setup.sh
```

Детали переносимости между платформами: `docs/codex_compat.md`.

## Быстрый старт

Напишите агенту запрос обычным языком, например:

> Собери карту сегментов и CJM для категории «средства от простуды», бренд — [название].
> Категория безрецептурная. Данных по отзывам пока нет. Нужно 2 устойчивых сегмента
> с путём пациента и вариантами ключевого сообщения.

или классический тест сообщений:

> Протестируй на синтетической панели 5 клеймов для [продукт] на сегментах [A] и [B].
> Шкала — готовность купить.

Эталон результата CJM-режима — пилотный прогон в `runs/cjm_hairloss_demo_20260710-0017/cjm_report.md` (демо-категория «средства от выпадения волос», вымышленный бренд).

## Что умеет и чего не делает

Шесть сценариев: ранжирование клеймов · скрининг концептов · сегментные реакции · аудиторная разведка · карта сегментов / AI CJM · конкурентная отстройка (проба знания брендов, карта территорий, switch-барьеры, тест отстроечных сообщений той же панелью). Панель принципиально **не делает**: Brand Lift, медиамикс, прогноз продаж, абсолютные «% купят», частоты/объёмы потребления, сенсорику, реальную долю рынка или switch rate — на такие запросы отвечает отказом с объяснением (это гейт, а не ограничение реализации).

Точность метода: SSR даёт **R=0,72 (~90% теоретического потолка)** на 57 валидационных опросах (потолок — повторяемость ответов самих людей) — это результат **оригинальной англоязычной конфигурации** метода; русская реализация панели валидируется собственным вычислительным гейтом шкалы (монотонность якорных наборов и rank-recovery на зафиксированном эмбеддере — см. `docs/embedder_ab_v13.md`) до собственного human-бенчмарка. Это не «90% точности»: надёжны относительные сравнения, не абсолютные цифры. Каждый прогон несёт встроенный самоконтроль (плацебо и пара-ловушка), каждый отчёт — несъёмный блок «Границы этого отчёта» и автоматическую проверку честности цифр и цитат (`scripts/cjm_lint.py`).

## Структура

```
SKILL.md                  # оркестрация: сценарии, гейты, поток
config.yaml               # провайдер LLM (agent/anthropic/openai/gigachat), эмбеддер, параметры
references/               # методология, якорные наборы (RU), шаблоны CJM-промптов и отчёта, дисклеймеры
scripts/                  # SSR-движок, генерация, CJM-конвейер, линтер честности, тесты
panel/segments/           # сегментные профили (клиентские наборы — только локально, см. .gitignore)
studies/                  # описания исследований
runs/                     # результаты прогонов (в репо — только эталонный пилот)
docs/                     # совместимость с Codex, ревью-отчёты
```

## Ограничения текущей версии

- Persona-режим: сегменты — качественная симуляция без калибровки на респондентских массивах (калибровка — следующая фаза).
- Без API-ключей генерация идёт в agent-режиме (`mode: exploratory` в манифесте: модель самодекларируется, temperature не контролируется, независимость прогонов сегментации — приближённая). С ключами (`ANTHROPIC_API_KEY` / `OPENAI_API_KEY` / `GIGACHAT_AUTH_KEY`) и валидированным стеком эмбеддера — режим `validated`.
- Русская шкала прошла собственный вычислительный гейт (монотонность/rank-recovery), но human-бенчмарк на российских данных впереди — до него все количественные обещания сознательно занижены; rank-recovery на малом банке парафраз помечен как ориентировочный (см. `docs/review_v1.3.md`).

Внутренний инструмент агентства. Не предназначен для представления синтетических данных как ответов реальных респондентов.
